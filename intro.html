Erlang

Erlang was first developed in the 1980's at Ericsson for programming telephone switches. It was released as open source in 1998.

Erlang is a very practical language. It isn't very functionally pure like Haskell. It is actually designed from the bottom up to build fault tolerant highly available systems. These design decisions impact the language profoundly.

Some of the core features are:

  • Concurrency: The language contains built in primitives (Erlang processes) for multithreading and concurrency. These are much lighter than threads in other languages and are used much more throughout Erlang. Processes can only communicate by passing messages and can never corrupt eachother's memory
  • Distribution: The highly concurrent nature of Erlang programs lends itself to distribution accross several computers on a network. The Erlang VM can handle distribution transparently behind the scenes.
  • Functional Style: The declarative nature of the language reduces the amount of code you need to write
  • The OTP Framework: The Open Telephony Platform is Erlang's standard library. Not only does it provide a bevy of modules for various tasks, it also has powerful libraries for common abstractions. You can implement common Erlang design patterns like servers, event handlers, finite state machines and event handlers all by simply implementing callback routines for these patterns. On top of all that are debugging, tracing, code coverage and static analysis tools built right into the standard erlang distribution.
  • Hot Code Reloading: The VM and standard libraries are designed to allow code changes to be brought online without disrupting any current clients or doing any rolling shutdowns

While you may be unfamiliar with Erlang as a language you've probably encountered a service that uses Erlang to run it's back end. Some examples

  • Facebook Chat: Built using Ejabberd an erlang XMPP implementation
  • Activision Blizzard: Runs Call of Duty servers on Erlang
  • What's App: Erlang Backend handles up to 10 billion messages a day. 2.8 Million connections to a single BSD server.
  • Distributed Databases: CouchDB, CouchBase and Riak are written in Erlang
  • RabbitMQ: An Erlang AMQP implementation
